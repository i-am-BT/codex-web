# Runtime Example

This directory documents the local writable state created under `runtime/`.

- All values are placeholders and contain no real credentials, session tokens, prompts, attachments, or user paths.
- Keep the real `runtime/` directory out of Git.
- Use mode `0700` for the real runtime and attachment directories.
- Use mode `0600` for JSON state, logs, and uploaded files.
- `port`, `server.pid`, and `server.log` are generated by the running service and normally should not be copied.
- Empty `images/`, `files/`, and `backgrounds/` directories show the expected upload layout.

The server creates missing runtime files and directories automatically.
